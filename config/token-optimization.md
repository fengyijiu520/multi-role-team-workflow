# Token Optimization Strategy
这是所有角色必须遵守的Token优化策略，每次任务前必须读取，严格执行。

## 核心优化规则
1. **时间戳优化**：使用mtime检查避免重复读取未变化的文件，97%场景实现0-Token消耗
2. **模型匹配优化**：
   - 纯体力活（格式转换、信息提取、简单检查）：使用Haiku模型，节省70-80% Token
   - 逻辑分析（问题诊断、方案设计、代码审查）：使用Sonnet模型
   - 创新设计（架构决策、战略规划、深度思考）：使用Opus模型，必须用户确认后才能使用
3. **Skill优先机制**：有专业技能可用时，必须优先使用Skill，禁止重复造轮子
4. **任务分解**：复杂任务必须拆分为子任务，每个子任务选择最合适的模型执行
5. **Git安全**：所有Git写操作必须用户确认，禁止自动提交、推送或删除

## 模型选择参考表
| 任务类型 | 推荐模型 | Token节省 | 适用场景 |
|----------|----------|-----------|----------|
| 数据提取/格式转换 | Haiku | 70-80% | JSON格式化、提取日志、统计数据 |
| 简单验证/检查 | Haiku | 70-80% | 格式校验、一致性检查 |
| 逻辑分析/策略制定 | Sonnet | 基准线 | Bug诊断、方案设计、代码分析 |
| 复杂创新/战略决策 | Opus | +3-5倍 | 架构设计、商业策略（需用户授权） |
