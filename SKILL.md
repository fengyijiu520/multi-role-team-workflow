# 多角色团队工作流
## 概述
该技能为智能体实现了结构化的团队协作系统，内置4个专业角色、强制安全审查、角色技能优先级，并且完整集成了 OpenClaw 框架的能力。它强制清晰的角色边界、自动化任务分配、端到端安全检查，以及不可跳过的物理检查点，确保行为可靠。

**核心要求**：所有角色的任务都必须通过子代理来执行，禁止AI自己扮演所有角色，禁止跳过任何流程和检查点，必须严格按照工作流执行。

核心原则：**清晰的角色边界 + 强制工程化工作流 + 子代理隔离 + 技能优先级 = 可靠、安全、高质量的多智能体协作。**

## ✨ 能力
1. **完整的软件工程工作流**：集成了 Superpowers 的7大核心技能，串联成完整的开发流水线
2. **自动依赖检查与安装**：自动检测 openclaw 目录，检查依赖技能，未安装时自动从服务器下载
3. **动态角色管理**：支持查看、新增、删除角色，内置角色保护，支持三种角色类型下载
4. **细粒度任务分解**：将任务拆解为2-5分钟的小任务，每个任务都有精确的文件路径、完整代码和验证步骤
5. **隔离工作区**：所有角色执行任务时，都会在新分支创建隔离工作区，保护主工作区
6. **子代理驱动开发**：每个任务都派遣全新的子代理，执行两阶段审查，避免上下文污染
7. **强制TDD流程**：开发工程师强制执行红绿重构流程，确保代码质量
8. **技能优先级加载**：每个角色优先使用自己的技能库，然后是主技能的，最后是全局的
9. **Token优化** : 使用缓存机制避免重复读取，节省67-85%的Token消耗
10. **Git安全保护** : 禁止自动提交，所有写操作必须用户确认
11. **受限办公自动化** : 为文员角色提供安全的Excel生成能力
12. **自动更新**：支持一键检查更新，自动下载并替换文件

## 适用场景
✅ 推荐使用：
- 你需要在专业AI角色之间进行结构化的分工
- 你需要完整的软件工程工作流，确保代码和文档的质量
- 你需要对所有用户/智能体交互进行强制安全审查
- 你需要自动化的任务拆解，并将任务分配给最合适的角色
- 你想要为你的独特工作流创建自定义AI角色

❌ 不推荐使用：
- 你正在处理简单的单任务，不需要分工
- 你不需要对交互进行安全审查
- 你更偏好非结构化的单智能体工作模式

## 核心工作流
```Plain Text
用户输入 → 安全网关审查 → 项目经理角色 → 需求澄清 → 角色自动识别 → 任务拆解 → 任务分配 → 角色执行 → 安全网关审查 → 用户输出
```

## 内置角色
我们将每个角色都实现为独立的技能，支持动态加载和发现，每个角色都有自己的独立技能库：

### 1. 项目经理 (`manager` 角色)
**身份** : 项目/团队管理者，团队的中央协调者 
**职责** :
- 接收经过验证的用户任务
- 调用技能完善需求，逐步和你确认
- 自动识别所有已启用的角色技能
- 拆解任务，评估任务权重，分配给最合适的角色
- 向你汇报任务进度和完成情况
- **内置角色，不可删除**

### 2. 安全网关 (`security_gateway` 角色)
**身份** : 交互安全审查者，整个系统的安全守护者 
**职责** :
- 审查所有用户输入和角色输出
- 执行安全规则，拦截违规内容
- 拦截跳过流程、越权操作的行为
- **内置角色，不可删除**

### 3. 开发工程师 (`developer` 角色)
**身份** : 技术实现者，编码专家 
**职责** :
- 仅接收来自项目经理的任务
- 使用技能完成开发，优先使用自己的技能库
- 编写符合规范的代码，执行测试
- 完成后将结果提交给安全网关审查

### 4. 文员 (`clerk` 角色)
**身份** : 文档与行政专员，办公管理者 
**职责** :
- 仅接收来自项目经理的任务
- 处理文档归档、行政事务
- 使用预批准的脚本安全生成 Excel 文档

## 角色管理
你可以使用 `./team-cli.sh` 来管理角色：
- 查看所有角色，区分内置和用户角色
- 新增角色：支持纯净/基础/完整三种类型，从服务器自动下载
- 删除角色：仅能删除用户自定义角色，内置角色受保护

## 技能加载优先级
每个角色执行任务时，技能的查找顺序为：
1.  **角色本地技能库**：`./roles/<角色名>/skills/`
2.  **主技能本地依赖**：`./skills/`
3.  **全局 OpenClaw 技能库**：`openclaw/skills/`

## 依赖的子技能
该技能会自动检查依赖技能，未安装时会询问你是否从服务器下载：
- `superpowers` 核心框架
- 其他依赖技能

## 更新功能
你可以使用 `./team-cli.sh` 中的更新选项，自动检查版本，下载更新包，一键更新到最新版本。

## 常见错误规避
1. **跳过安全审查** : 绝不要绕过网关 —— 它会保护你免受不安全的内容和操作的影响
2. **直接向专业角色发送指令** : 所有任务必须通过项目经理分配，保持工作流结构化
3. **自己扮演所有角色** : 必须通过子代理执行各个角色的任务，禁止跳过流程
4. **修改安全规则** : 绝不要修改安全规则文件，这会破坏整个安全系统
5. **删除内置角色** : 经理和安全网关是核心内置角色，不允许删除
